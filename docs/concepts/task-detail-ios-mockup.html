Task Detail iOS — Atual vs Proposto

Comparação visual da tela de detalhe de tarefa (TaskDetailView.swift). A coluna da esquerda reflete o layout após a mudança recente das subtarefas. A direita propõe refinamentos de hierarquia, consistência e densidade, sem alterar funcionalidades.

Problemas no atual

  • Cabeçalhos inconsistentes: Subtarefas em 15px bold; Comentários em 12px cinza.
  • Notas em mini-card: caixa separada que compete visualmente com o card de metadados.
  • Metadados redundantes: linhas "Nenhuma" + pílulas vazias duplicam o mesmo affordance.
  • Subtarefas pobres: só título + lixeira; sem chips de data/etiqueta já disponíveis no modelo.
  • Lixeira sempre visível: ação destrutiva em destaque; context menu já existe.
  • Comentários fragmentados: cada item é um card solto; input fora do grupo.

Melhorias propostas

  • Sistema de seção único: mesmo header (título + badge + chevron) para Subtarefas e Comentários.
  • Notas inline: texto secundário abaixo do título, sem caixa até haver foco/conteúdo longo.
  • Metadados enxutos: só linhas preenchidas; campos vazios só nas pílulas.
  • Rows de subtarefa ricas: anel com cor de prioridade + chips de vencimento/etiqueta.
  • ⋮ no lugar da lixeira: menos ruído visual; excluir continua no menu de contexto.
  • Comentários em card: lista + composer no mesmo container, espelhando subtarefas.

Princípios (Impeccable / product)

  • Consistência: mesmo vocabulário visual entre seções (cards + headers).
  • Accent só para ação: ciano reservado a botões, foco e adicionar.
  • Densidade controlada: metadados compactos; subtarefas com 2ª linha só quando há dados.
  • Sem cards aninhados: um nível de surface por bloco, não caixa dentro de caixa.
  • Contraste legível: corpo secundário ≥4.5:1 sobre fundo escuro.
