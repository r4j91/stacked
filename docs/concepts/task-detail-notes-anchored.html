Notas no detail — campo inline vs painel ancorado

Mockup só visual (sem código do app). Compare o campo atual sob o título com a linha “Notas ›” que abre um painel no estilo dos menus de prioridade/etiquetas — mas com textarea grande, porque descrição não é lista de opções.

Listas / scroll / swipe / animação de concluir — fora de escopo

Problema

O TextField inline sob o título fica apertado, cresce o sheet e compete com o card de metadados.

Proposta

Linha compacta sob o título (preview ou “Adicionar notas…”). Toque abre painel ancorado com área de escrita generosa.

Fluidez

Não muda o scroll das listas (já usam preview read-only). No detail: menos altura viva e meta card mais estável.
